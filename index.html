How come rule builder does not just call compile for me? Rule builder assumes a few things when it comes to calling compile: a. The developer knows best when to call compile. Compiled rules should usually be backed by a lazy loaded cache to avoid calling compile. b. Compile will sometimes invoke reflection, sometimes not. For any operator but 'Equals' on a string comparison, reflection will be used to build the expression. None of the numeric operators invoke reflection when compile is called. None of the claims based rules invoke reflection. c. mix-ins are only suppoorted against expressions not compiled rules. you can use mix-ins all you want and then you call compile on the newly composed rule. 2.
